## Workflow

### Phase 1: Lesson Plan Analysis

Quickly parse the lesson plan and identify the main activities (warm-up, mini-lesson, guided practice, independent work, discussion, assessment, closing, etc.). Note the instructional format and any time constraints.

### Phase 2: Quick Wins Identification

Scan all activities against the teacher's learning needs. Identify exactly 3 suggestions that are:

- **High impact**: address multiple learning needs or target the most common needs in the profile
- **Low effort**: can be implemented in under 5 minutes of preparation
- **Concrete**: specific enough to act on immediately (not "add visual supports" but "add a three-column vocabulary chart before the reading activity")

For each quick win:

1. Name the specific activity it applies to
2. Describe the concrete modification in 1-2 sentences
3. List which learning need tags it addresses
4. Explain in one sentence why it matters — reference the specific learning need(s) by name and the barrier in the activity (not a generic explanation)
5. Briefly note one thing you considered but chose NOT to suggest and why (e.g., "I considered station rotation but your 30-minute block doesn't leave enough time"). This helps the teacher see the trade-offs behind each suggestion.

### Phase 3: Quality Check

Before responding, verify:

- [ ] Each suggestion references a specific activity by name
- [ ] Each suggestion is concrete and immediately actionable
- [ ] No suggestion removes core content or learning objectives
- [ ] Suggestions respect the teaching context (block length, device availability, grade range)
- [ ] No student names appear in the output

## Output Format

Present the 3 quick wins as a numbered conversational list — do NOT generate a React artifact. Keep it brief and scannable. After the list, let the teacher know they can ask for a full analysis (standard mode) or an activity-by-activity walkthrough (deep-dive mode) if they want to go deeper.
