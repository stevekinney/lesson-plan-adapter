## Workflow

Follow these phases in order. Complete each phase fully before moving to the next.

### Phase 1: Lesson Plan Analysis

Parse the lesson plan and identify discrete activities (warm-up, mini-lesson, guided practice, independent work, discussion, assessment, closing, etc.). If the plan has clear sections, use those boundaries. If loosely structured, infer 3-6 logical phases. If very short, use a single activity called "Full Lesson." Note the instructional format, time allocations, and learning objectives for each activity.

### Phase 2: Needs Mapping

For each activity, identify which learning needs from the teacher's profile are relevant. An activity may be relevant to multiple needs. A need may be relevant to multiple activities. Only map genuine connections — not every need applies to every activity.

### Phase 3: Adaptation Generation

For each activity, generate specific, actionable adaptation suggestions. Each suggestion must:

- Reference the specific activity by name
- Propose a concrete modification (not a category of modification)
- Include a rationale explaining WHY this modification fits this classroom — reference the specific learning need(s) by name and the barrier in the activity. For example: "Students with text-read-aloud needs will struggle with the paragraph-dense lab instructions, so adding numbered steps with bold action words removes the reading barrier without changing the content."
- Be tagged with the learning need(s) it addresses
- Be classified by effort level (`quick-win`, `moderate`, or `deeper`)
- Be classified by UDL category (representation, expression, or engagement)
- Include implementation detail a teacher could act on immediately

Aggregate where possible: if multiple needs point toward the same modification, combine them into one suggestion with multiple tags rather than repeating similar suggestions.

### Phase 4: Transition Analysis

Examine transitions between activities. Identify where students with structured transition needs, sensory regulation needs, or attention-related needs might struggle. Generate transition notes with specific suggestions for smoothing each transition.

### Phase 5: Quick Wins Extraction

From all suggestions, select 2-3 that are high-impact and low-effort as featured quick wins for the banner.

### Phase 6: Quality Verification

Before generating the output, verify:

- [ ] Every suggestion references a specific activity by name
- [ ] Every suggestion is concrete and actionable (not "provide visual supports" but "replace the text-only definitions with a three-column vocabulary chart")
- [ ] No suggestion removes core content or learning objectives
- [ ] Suggestions respect the teaching context (block length, device availability, grade range)
- [ ] At least one quick win exists
- [ ] No student names appear in the output
- [ ] All tags used are from the teacher's profile
- [ ] Time impact is noted for suggestions that add duration
